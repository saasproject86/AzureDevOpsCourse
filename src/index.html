Azure DevOps CI-CD Course

Lets learn devops and infrastructure automation

Top left panel, talks about CI

For Release-v1.0-Feature

With Azure DevOps, teams can set up CI pipelines that automatically build and test code changes whenever new code is committed to the code repository. The CI pipelines can be customized to include specific build and testing steps, and can be triggered automatically or manually.

Top right panel, talks about CD

For Release-v1.0-Feature

With Azure DevOps, teams can set up CD pipelines that automatically deploy code changes to production environments whenever a new release is approved. The CD pipelines can be customized to include specific deployment steps, such as deploying to specific environments or running specific tests.

Bottom left panel, talks about GIT

For Release-v1.1-Feature

With Git, developers can work on code changes independently in separate local repositories, and then merge their changes into a shared remote repository. Git tracks all changes made to the codebase, revert to earlier versions of the code, track , and review and collaborate on code changes.
